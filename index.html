Our Project

The Council answers numerous Freedom of Information requests every year. From the number of parking tickets issued in specific areas to maintenance of Council owned buildings, FOI covers all information held by the public authority.

As of right now all requests are handled manually by officials. Due to requests being redirected internally to the respective department this process can take up to two months. Often times the request has been made before, resulting in extra work done by the civil servant.

The requests are answered using pdf documents. In the future the council wants to move away from this formal answering method and provide short, precise answers. This project helps exploring new ideas in this area.

Given the above areas to improve. We are working on a chatbot that analyzes FOI requests and determines whether these particular questions have been answered before. In addition a user can make a new FOI request through the chatbot. Also, we will start exploring a new approach to answering the requests in a single question, single answer format.

Our Solution

Our solution consists of two main parts: a chatbot and an nlp endpoint. The user interacts with the chatbot on a website and can ask FOI requests. The question is then to the flask endpoint where it is processed and searched for any matching documents. Additionally, the user can make new requests through the bot that are send to an email address to be processed by hand.

Key Features

  • To provide an endpoint where users could submit their FOI requests and receive instant feedback if their request is similar to a query that has been previously answered.
  • To provide the user with the capability of emailing their request directly to Camden through the chatbot.

Project Goals

The goal of this project is to explore chatbot possibilities using the given FOI request data. This project consists of a stage of data exploration and analysis. Also, a chatbot endpoint will be explored. There a different platforms and types of chatbots to consider. In the end this project should be seen as a prototype of a potential chatbot for FOI requests. This stepping stone could then be taken further.



MoSCow Requirements

Must Have
  • Submit FOI requests through the chatbot
  • Provide a chatbot endpoint that can be hooked up with some chat platform
Should Have
  • Categorize new requests
  • Provide information on previously answered questions
  • Access bot through services such as skype or facebook messenger
  • Analyse how similar the request is to previously asked questions
Could have
  • Gather statistics on the conversations
  • Answer new questions from knowledge gathered from previous questions
  • Request Validation
